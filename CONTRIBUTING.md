# Contributing

## Setup

```bash
# install dependencies
npm install

# start the app
npm run dev
```

## Before opening a PR

```bash
# run tests
npm test

# build the app
npm run build
```

## Project rules

1. Do not commit proprietary game assets.
2. Keep exported example data repo-safe.
3. Prefer room-level editing over raw tile editing in v1.
4. Keep validation messages specific and actionable.

## Good contribution targets

- Better DS1 metadata parsing
- Stronger generator heuristics
- More export safety checks
- Better room tagging UX
- Better docs from real mapping workflows
